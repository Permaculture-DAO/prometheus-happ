# Canonical Documents

This directory lists high-level canonical references for the Prometheus ecosystem.

## Order of authority

1. Canonical white paper release
2. Prometheus Canon - 12 Pages
3. Repository README and CONTRIBUTING files
4. Repository-bounded implementation artifacts

## Interpretation rule

These documents must be read together, but they do not have the same function.

- The white paper establishes the master constitutional architecture.
- The Canon provides a compressed, high-density constitutional overview.
- Repository README and CONTRIBUTING files define repository-specific scope and discipline.
- Runtime code implements bounded pilot behavior and must not silently redefine constitutional meaning.

## Current pilot discipline

The current pilot remains bounded.

Excluded from current pilot scope include:

- frontend / UI
- token market layer
- full future financial layer
- securitization expansion
- sensor hardware deployment
- broader platform redesign
